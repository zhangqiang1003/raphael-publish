# Raphael Publish - 产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品定位

**Raphael Publish** 是一款专为**微信公众号创作者**打造的现代 Markdown 排版引擎工具，致力于解决内容创作者在公众号排版过程中的痛点，提供"所见即所得"的一站式排版解决方案。

### 1.2 目标用户

- **核心用户**：微信公众号运营者、内容创作者
- **扩展用户**：技术博主、产品经理、新媒体编辑等需要发布图文内容的群体
- **使用场景**：需要将 Markdown/富文本内容发布到微信公众号，追求高效、美观的排版效果

### 1.3 核心价值主张

1. **零学习成本**：无需掌握 Markdown 语法，粘贴即用
2. **一键发布**：从编辑到公众号发布，只需一次复制粘贴
3. **专业美观**：30 套精心设计的主题，告别同质化
4. **微信兼容**：深度适配微信公众号编辑器，样式精准还原

## 二、用户需求分析

### 2.1 用户痛点

#### 痛点 1：富文本内容迁移困难
- **场景**：从飞书、Notion、Word 等平台编写的内容无法直接用于公众号
- **问题**：手动复制粘贴会丢失格式或带入大量冗余样式
- **现有方案缺陷**：需要手动调整格式，效率低下

#### 痛点 2：Markdown 学习门槛
- **场景**：非技术人员不会写 Markdown
- **问题**：传统 Markdown 编辑器需要学习语法
- **现有方案缺陷**：学习成本高，推广困难

#### 痛点 3：公众号排版工具同质化
- **场景**：需要差异化、品牌化的视觉风格
- **问题**：现有工具模板单一，缺乏设计感
- **现有方案缺陷**：白底黑字模板泛滥，无法满足个性化需求

#### 痛点 4：图片外链限制
- **场景**：使用网络图片时提示"此图片来自第三方"
- **问题**：微信公众号对外链图片有限制
- **现有方案缺陷**：需要手动下载上传，流程繁琐

#### 痛点 5：多图排版困难
- **场景**：需要并排展示多张图片（如对比图、组图）
- **问题**：微信编辑器不支持 flex 布局
- **现有方案缺陷**：多图会被强制换行，无法实现网格布局

#### 痛点 6：预览不直观
- **场景**：无法看到在手机上的真实显示效果
- **问题**：桌面编辑和手机展示存在差异
- **现有方案缺陷**：需要在手机上反复查看，效率低

### 3.2 功能详细设计

#### 功能 1：魔法粘贴 (Magic Paste)

**用户故事**：
> 作为一名内容创作者，我希望直接从飞书/Notion/Word 复制内容后粘贴就能自动转换为干净的 Markdown，这样我就不需要学习 Markdown 语法或手动清理格式。

**功能描述**：
- 监听用户粘贴操作（Ctrl/Cmd + V）
- 智能识别剪贴板内容类型（富文本、图片、纯文本）
- 自动清洗富文本，转换为标准 Markdown
- 支持截图/图片直接粘贴，自动插入 Markdown 图片语法

**技术实现要点**：
1. 使用 Turndown 库进行 HTML → Markdown 转换
2. IDE 格式化代码识别（避免误处理代码块）
3. Markdown 内容检测（已是 Markdown 则不重复处理）
4. 图片文件转 Base64 并生成 Markdown 语法

**验收标准**：
- ✅ 从飞书复制带标题、列表、粗体的内容，粘贴后为纯净 Markdown
- ✅ 粘贴截图后自动生成 `![图片](data:image/...)` 语法
- ✅ 从 IDE 复制代码块时保留原始格式，不转换为 Markdown
- ✅ 已经是 Markdown 的内容粘贴后保持不变

**成功指标**：
- 魔法粘贴使用率 > 80%
- 粘贴后手动调整率 < 5%

---

#### 功能 2：一键复制到公众号 (One-Click Copy)

**用户故事**：
> 作为一名公众号运营者，我希望点击一个按钮就能将排版好的内容完美复制到公众号后台，不用担心图片外链或样式错乱的问题。

**功能描述**：
- 将渲染后的 HTML 进行微信兼容性处理
- 所有外链图片自动转为 Base64 编码
- Flex 布局转换为 Table 布局（微信兼容）
- 列表项扁平化处理，防止嵌套错误
- 强制字体继承，避免样式被覆盖
- CJK 标点符号防断行处理

**技术实现要点**：
1. DOM Parser 解析 HTML
2. 图片异步转 Base64（Promise.all 并发处理）
3. 图像网格 `<p class="image-grid">` → `<table>` 转换
4. 内联样式强制注入（针对微信编辑器特性）
5. 零宽空格/Unicode 字符处理（防止换行断裂）

**验收标准**：
- ✅ 复制后的内容粘贴到公众号后台，样式完全一致
- ✅ 网络图片正常显示，无"第三方图片"报错
- ✅ 多图并排布局在微信中正确显示，不换行
- ✅ 列表、表格在微信中结构完整，不塌陷
- ✅ 特殊标点符号（如 `：`）不与前文分离

**成功指标**：
- 复制成功率 > 99%
- 用户二次调整率 < 3%

#### 功能 4：多端实时预览 (Multi-Device Preview)

**用户故事**：
> 作为一名细心的作者，我希望在编辑时就能看到文章在手机、平板上的真实效果，避免发布后才发现排版问题。

**功能描述**：
- 三种预设设备尺寸：
  - 手机：480px（模拟微信真实阅读宽度）
  - 平板：768px（横屏阅读场景）
  - 桌面：自适应（完整宽屏预览）
- 实时渲染，编辑与预览同步
- 支持双向滚动同步（Editor ↔ Preview）

**技术实现要点**：
1. 响应式布局（Tailwind CSS Grid）
2. 滚动位置比例同步算法
3. 防抖处理，避免滚动冲突
4. 移动端 Tab 切换（编辑/预览模式）

**验收标准**：
- ✅ 切换设备视图时，预览区域宽度立即改变
- ✅ 编辑区输入内容，预览区实时更新（延迟 < 100ms）
- ✅ 开启滚动同步后，两边滚动位置保持比例一致
- ✅ 移动端下可通过 Tab 切换编辑/预览模式

**成功指标**：
- 预览功能使用率 > 90%
- 滚动同步开启率 > 60%

**用户故事**：

**功能描述**：
- 自动识别连续的单图段落

- 支持圆角、阴影、边框等美化效果

**技术实现要点**：
1. Markdown 解析后处理（Post-processing）
2. DOM 节点合并算法
3. Flex → Table 布局转换（复制到公众号时）
4. 图片样式统一化（圆角、阴影、边框）

**验收标准**：
- ✅ 连续输入两张图片 Markdown 语法，渲染为并排布局
- ✅ 复制到公众号后，多图仍然并排显示
- ✅ 图片网格具有统一的视觉样式（圆角、阴影）
- ✅ 奇数张图片时，最后一张单独占一行

**成功指标**：
- 多图网格使用率 > 40%
- 微信中多图显示成功率 100%

### 3.3 辅助功能

#### 功能 7：滚动同步 (Scroll Sync)

**设计目标**：提升长文编辑体验，减少手动切换视线

**实现逻辑**：
- 计算源滚动容器的最大滚动距离
- 计算目标滚动容器的最大滚动距离
- 按比例同步滚动位置
- 50ms 延时释放锁，避免抖动

**交互细节**：
- 工具栏提供开关按钮（链接图标）
- 开启状态：蓝色高亮
- 关闭状态：灰色

---

#### 功能 8：响应式布局 (Responsive Design)

**设计原则**：Mobile First，渐进增强

**断点定义**：
- 移动端：< 768px
- 平板：≥ 768px
- 桌面：≥ 1024px

**适配策略**：
- 移动端：单栏布局，Tab 切换编辑/预览
- 桌面端：双栏布局（黄金分割 38.2% : 61.8%）
- 工具栏：移动端分两行，避免按钮被挤出

### 5.2 核心数据流

```

    ↓

    ↓
[Markdown 解析器 markdown-it]
    ↓
基础 HTML
    ↓
[主题应用 applyTheme]
    ├─→ 遍历主题样式表
    ├─→ 注入内联样式
    └─→ 图片网格处理
    ↓

    ↓
[微信兼容处理 makeWeChatCompatible]
```
